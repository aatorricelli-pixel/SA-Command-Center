[ Live Data Integration Roadmap ]

Source and API Plan

Backend required for protected feeds, RSS, scraping and API keys

Weather

SAWS warnings via AfriGIS OAuth2 API. OpenWeatherMap can provide fallback alerts and raster layers.

Traffic / Rail

TomTom or HERE for live incidents and flow. i-Traffic, PRASA, Metrorail and Gautrain need backend collectors.

Infrastructure

EskomSePush API for load-shedding. Joburg Water and City Power require scraping/OCR/PDF parsing jobs.

Finance

TradingView widgets for charts. Twelve Data or JSE data feeds for controlled JSE, forex, commodity and crypto prices.

Conflict / Security

ACLED API for protests, riots and conflict events. GDELT can supplement broad news/event monitoring.

Maritime / Aircraft

MarineTraffic, AISHub or Spire for AIS. OpenSky or ADS-B Exchange for aircraft. Use public, delayed or authorized feeds only.

Map Layers

MDB shapefiles and OpenStreetMap/Overpass queries for provinces, municipalities, roads, rail, ports, airports and bases.

News

News24, Daily Maverick, SABC and Google News RSS should be fetched by backend and converted to JSON.

Backend

Recommended: Supabase Edge Functions with Postgres/PostGIS, or Render Node/Python service for cron jobs, caching and secure keys.
